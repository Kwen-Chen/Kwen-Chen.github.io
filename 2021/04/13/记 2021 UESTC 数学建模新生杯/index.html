记 2021 UESTC 数学建模新生杯

2021 数学建模新生杯落下帷幕，在两天半的头脑风暴后，我想把这一切，包括所想所感所学，全部记录下来，作为 CK’s blogs 的第一篇正式的博客。希望这是一个美好而有意义的记录。

准备比赛

​ 电子科技大学的数学建模文化一直很浓厚，作为一个什么比赛都想试试的人来说，数学建模当然要参加。在第一学期已经很幸运地寻找到两位志同道合的队友，并成功参赛第一学期美赛模拟赛的基础上，我对这次新生杯比赛已经不再陌生，甚至会有一些期待。

​ 期待比赛意味着必须要对这次比赛做好万足准备，对于这次新生杯比赛，我又做了哪些合适的准备，还有哪些需要改进的地方呢？

准备

  1. 提前了解待参加数学建模比赛类别（如国赛、美赛），并做好 LaTeX 模板准备，对于LaTeX 模板，可以去 GitHub 上寻找，在这里列出本次新生杯所用的国赛模板 GitHub 地址：

    GitHub 上国赛 LaTeX模板

    拿到LaTeX 模板一定要在自己的编译器上做好调试，出现了任何问题都有时间及时解决，不能把比赛的宝贵时间浪费在无用的调试 LaTeX 代码上，这肯定是没有意义的。一般提供的模板是以模板使用说明的方式提供的，所以需要备份两份，一份作为使用说明，一份准备套入写作模板。

    本次比赛就在这方面尝到了甜头，一路写下来非常顺利，即使我已经很久没有用过LaTeX了！

  2. 提前学习了解各大模型及其算法程序，这当然是一个数学建模人必修的功课，但在比赛前昔，我们可以粗略地浏览一遍各大类型模型，比如规划模型类、评价模型类、预测模型类等等，


    /images/loading3.gif" data-original="

    做到心中有数，拿到比赛题目时就可以直到这大概属于那个类型的题目，在那个类型的模型中，再逐一去具体地找合适的模型。在这里也列出我的一位校友学长的个人博客，他的博客不仅是我开发个人博客的源动力，这篇数学建模模型总结也具有相当高的质量，下面这篇博客也正是这位学长的：

    geminiplanet

  3. 我至今都相信，建模编程占五分，写作一家就占五分，可能会更多，一篇好的模型，倘若写的稀巴烂也没有任何意义，但是一个连自己都无法说服的模型，写作手只要能圆好，一样有可能可以受到青睐。那写作如何准备呢？我在比赛前一天，研究了一下历年来的国赛一等奖论文，尤其是被我们学校数模教练推荐的获奖论文，从这些论文里，你一定可以提炼出一套写作模板。好吧，我打算把我的写作模板给大家列出来：
